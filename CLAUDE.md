## Project Overview

ATP CRM is a custom CRM for Tracetech PCB Design Services. It manages the full pipeline: enquiries → quotes → jobs, plus daily hours tracking and data import. It is a monorepo with a Node/Express backend and a React/Vite frontend.

## Commands

Run from the repo root:

```bash
# Start both backend and frontend in dev mode
npm run dev

# Backend only (nodemon, port 3001)
npm run dev:backend

# Frontend only (Vite, port 5173)
npm run dev:frontend

# Production build (frontend dist)
npm run build

# Production start (backend serves built frontend)
npm start
```

Install dependencies separately for each workspace:
```bash
cd backend && npm install
cd frontend && npm install
```

No tests or linter are configured.

## Environment

Backend reads `.env` in `backend/`. Key variables:
- `MONGO_URI` — defaults to `mongodb://localhost:27017/atp_crm`
- `SESSION_SECRET` — defaults to `atp_crm_dev_secret`
- `PORT` — defaults to `3001`

## Architecture

### Backend (`backend/`)

- **Entry**: `server.js` — connects to MongoDB, runs `seedDatabase()`, then starts Express on port 3001.
- **App**: `src/app.js` — mounts CORS, MongoDB-backed sessions (`express-session` + `connect-mongo`), and all routes under `/api/*`.
- **Auth**: Cookie-based sessions. `req.session.userId` and `req.session.role` are set on login. `src/middleware/auth.js` exports `requireAuth` and `requireRole(...roles)`.
- **Seed**: `src/routes/seed.js` runs on every startup — idempotently creates default admin accounts and seeds sample data on an empty DB.

**Models** (Mongoose, all in `src/models/`):
- `User` — roles: `admin | manager | designer`. Passwords hashed via bcrypt pre-save hook; `.toSafe()` strips the password field.
- `Enquiry` — lead/contact record. Statuses: `new → contacted → quoted → converted → closed`.
- `Quote` — optionally linked to an Enquiry (`enquiryId`). Statuses: `pending | approved | rejected | converted`.
- `Job` — core work record linked to a Quote (`quoteId`). Has an `atpNumber` auto-generated by `src/utils/atpNumber.js` as `ATP-YY-NNN` (2-digit year + sequential count of jobs that year). Tracks `jobOwner`, `designer`, `status`, and `paymentStatus`.
- `DailyHours` — time log entries optionally linked to a Job (`jobId`).

### Frontend (`frontend/src/`)

- **Stack**: React 18, React Router v6, TanStack Query, Tailwind CSS, `sonner` for toasts.
- **Path alias**: `@/` → `frontend/src/`.
- **API client**: `api/client.js` — axios instance with `baseURL: /api`, `withCredentials: true`. A 401 interceptor auto-redirects to `/login` (except on `/auth/me`).
- **Auth state**: `context/AuthContext.jsx` — exposes `user`, `login`, `logout`, and `can(...roles)` helper used for role-based UI gating.
- **Routing** (`App.jsx`): All routes live under `AppLayout` behind a `ProtectedRoute`. Role-gated: Quotes/Import require `admin|manager`; Data/Users require `admin`.
- **API modules** (`api/`): One file per resource (`auth`, `enquiries`, `quotes`, `jobs`, `hours`, `users`, `stats`, `data`), all using the shared client.
- **UI components**: `components/ui/` — primitives (Button, Input, Dialog, Badge, Select, Textarea, Spinner). `components/shared/` — composed pieces (PageHeader, StatCard, FormField, EmptyState, ConfirmDialog).

### Data Flow

Enquiries can be promoted to Quotes; Quotes can be converted to Jobs. The `atpNumber` is generated at Job creation time and is the primary human-readable job identifier.

### Production

When `NODE_ENV=production`, Express serves the Vite build from `frontend/dist` and falls back to `index.html` for all non-API routes. A `Dockerfile` is present for containerized deployment.
